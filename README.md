# CS-305 Portfolio

## Reflection

### Briefly summarize your client, Artemis Financial, and its software requirements.

Artemis Financial is a financial services company that stores sensitive customer information, including retirement plans, investments, insurance accounts, and savings data. The company needed to improve the security of its software application by protecting customer data, securing communications, and identifying vulnerabilities that could expose information to attackers.

### What did you do well when you found your client’s software security vulnerabilities?

I did well identifying vulnerabilities through both manual review and static testing. I was able to recognize issues such as outdated dependencies, weak input validation, missing authentication, insecure API access, and missing HTTPS enforcement. Coding securely is important because it protects sensitive information, reduces the risk of cyberattacks, and helps build trust between a company and its customers.

### Which part of the vulnerability assessment was challenging or helpful to you?

The most challenging part was understanding the different vulnerabilities and learning how to interpret dependency-check results and CVE reports. However, this was also one of the most helpful aspects because it showed me how real-world security issues are discovered and documented.

### How did you increase layers of security? In the future, what would you use to assess vulnerabilities and decide which mitigation techniques to use?

I increased security by recommending updated dependencies, HTTPS and TLS encryption, input validation, stronger authentication, secure error handling, and role-based authorization. In the future, I would continue using OWASP Dependency-Check, vulnerability scanners, and secure coding guidelines such as OWASP recommendations to identify and mitigate risks.

### How did you make certain the code and software application were functional and secure?

After making security improvements, I reviewed the code and vulnerability reports to ensure that the application remained functional while reducing security risks. I also used testing and dependency scans to verify that new vulnerabilities were not introduced.

### What resources, tools, or coding practices did you use that might be helpful in future assignments or tasks?

I used the OWASP Dependency-Check Maven plug-in, CVE and NVD databases, secure coding principles, and vulnerability assessment techniques. These tools and practices will be valuable in future software development and cybersecurity projects.

### What might you show future employers from this assignment?

I would show future employers my completed vulnerability assessment report because it demonstrates my ability to identify security risks, analyze software vulnerabilities, and recommend practical solutions to improve application security.
